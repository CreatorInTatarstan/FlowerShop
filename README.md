# 🌸 Цветочная лавка — Информационная система интернет-магазина цветов

Информационная система интернет-магазина цветов на PHP 8 + MySQL 8. Дипломный проект.

## ✨ Реализованный функционал

### Клиентская часть
- 🏠 Главная страница с категориями и популярными букетами
- 🔍 Каталог товаров с фильтрами (категория, цена, повод) и сортировкой
- 🌹 Детальная карточка товара с составом, размером, фото
- ⭐ Система отзывов и рейтингов (с модерацией)
- 🛒 Корзина с управлением количеством и промокодами
- 💳 Оформление заказа с выбором даты, времени и способа оплаты
- 👤 Личный кабинет с историей заказов и повтором заказа
- 🔐 Регистрация / авторизация с защитой паролей (`password_hash`)
- 📞 Информационные страницы (О нас, Доставка, Контакты)

### Админ-панель
- 📊 Аналитика: выручка, средний чек, график продаж за 14 дней, топ-товаров
- 📦 Управление заказами: статусы, печать накладных, фильтры
- 🌹 CRUD товаров с загрузкой изображений и контролем остатков
- 📁 Управление категориями (с подкатегориями)
- 👥 Управление пользователями и ролями (admin/manager/client)
- 🚚 Учёт поставщиков
- 📥 Учёт поставок с **автоматическим обновлением остатков на складе**
- 🎟️ Управление промокодами (бессрочные/срочные, активация/деактивация)
- ⭐ Модерация отзывов
- 📊 Экспорт отчётов в CSV (с поддержкой кириллицы для Excel)

### Безопасность
- ✅ CSRF-токены во всех формах
- ✅ Хеширование паролей через `password_hash` (bcrypt)
- ✅ Prepared statements (PDO) — защита от SQL-инъекций
- ✅ XSS-защита через `htmlspecialchars`
- ✅ Регенерация ID сессии при входе
- ✅ Защита папок `/includes/` и `/uploads/` от прямого выполнения PHP

## 📋 Требования

- PHP 8.0+ (с расширениями PDO, mbstring, gd)
- MySQL 5.7+ или MariaDB 10.3+
- OSPanel / OpenServer / XAMPP / любой Apache/Nginx
- Поддержка mod_rewrite (для красивых URL — опционально)

## 🚀 Установка на OSPanel

### Шаг 1. Размещение файлов

Скопируйте всю папку `flower-shop` в директорию доменов OSPanel:
```
C:\OSPanel\domains\flower-shop\
```

### Шаг 2. Создание домена

1. Кликните правой кнопкой по иконке OSPanel в трее
2. **Дополнительно → Добавить алиас** или просто перезапустите OSPanel — он сам создаст домен `flower-shop.local`
3. Откройте `http://flower-shop.local` — пока будет ошибка БД, это нормально

### Шаг 3. Создание базы данных

Откройте **phpMyAdmin** через меню OSPanel → Дополнительно → phpMyAdmin (либо `http://localhost/openserver/phpmyadmin/`):

1. Создайте новую базу данных:
   - **Имя:** `flower_shop`
   - **Сравнение:** `utf8mb4_unicode_ci`
2. Откройте созданную базу → вкладка **Импорт**
3. Выберите файл `database.sql` из папки проекта
4. Нажмите **Вперёд**

### Шаг 4. Настройка подключения

Откройте файл `includes/config.php` и проверьте параметры (для OSPanel по умолчанию подходит):

```php
define('DB_HOST', 'localhost');
define('DB_NAME', 'flower_shop');
define('DB_USER', 'root');
define('DB_PASS', '');     // По умолчанию в OSPanel пароль пустой
```

### Шаг 5. Запуск

Откройте в браузере: **http://flower-shop.local**

## 🛠️ Возможные проблемы

**Проблема: «Connection refused» / ошибка подключения к БД**
- Проверьте, что MySQL запущен в OSPanel
- Проверьте параметры в `includes/config.php`

**Проблема: «Domain not found» / `flower-shop.local` не открывается**
- Перезапустите OSPanel
- Проверьте, что папка проекта называется именно `flower-shop`
- Откройте `http://localhost/flower-shop/` если не используете алиасы

**Проблема: Картинки товаров не отображаются**
- В тестовых данных по умолчанию используется заглушка — это нормально
- При загрузке своих фото убедитесь, что папка `/uploads/` существует и доступна на запись

**Проблема: «Permission denied» на запись**
- Дайте права на запись папкам `/uploads/` и `/logs/`

## 🤖 Telegram-бот

### Возможности бота

**Для клиентов:**
- 🛍 Просмотр каталога с фото и ценами
- 🛒 Корзина прямо в Telegram
- 💳 Оформление заказа в боте (без перехода на сайт)
- 📦 Просмотр истории заказов и статусов
- 🔗 Привязка к аккаунту сайта — синхронизация заказов

**Для админов:**
- 🆕 Уведомления о новых заказах с возможностью менять статус прямо из чата
- ⭐ Уведомления о новых отзывах
- 👤 Уведомления о регистрации пользователей

**Уведомления клиентам:**
- ✅ Об оформлении заказа
- 🔄 О смене статуса (в работе → в доставке → выполнен)
- 🛒 О действиях с корзиной (добавление/удаление товаров)

### Установка бота

1. **Создать бота** через [@BotFather](https://t.me/BotFather):
   - `/newbot` → выбрать имя → получить токен

2. **Применить миграцию БД** в phpMyAdmin:
   - Откройте БД `flower_shop`
   - Импорт → выберите `database_telegram.sql`
   - (либо используйте `database_full.sql` который включает всё)

3. **Настроить в админке** → раздел «Telegram-бот»:
   - Bot Token — вставить токен от BotFather
   - Bot Username — например `FlowerShopBot` (без @)
   - Сохранить

4. **Локальная разработка (с HTTPS через ngrok):**
   ```
   ngrok http 80
   ```
   - Скопировать https-URL (например `https://abc.ngrok-free.app`)
   - В админке → Telegram: Webhook URL = `https://abc.ngrok-free.app/telegram/bot.php`

5. **Установить webhook:**
   - Открыть `/telegram/setup.php`
   - Нажать «getMe» — проверить токен
   - Нажать «Установить webhook»
   - Нажать «Установить команды»

6. **Узнать свой Telegram chat_id** (для админских уведомлений):
   - Написать боту [@userinfobot](https://t.me/userinfobot)
   - Скопировать ID
   - В админке → Telegram → «Chat ID администраторов» = указать ID через запятую

7. **Тест:** напишите `/start` своему боту в Telegram

### Структура `/telegram/`

- `bot.php` — webhook-handler (главный файл бота)
- `config.php` — настройки и хелперы
- `api.php` — класс работы с Telegram Bot API
- `notifier.php` — функции отправки уведомлений
- `setup.php` — установка webhook и команд

### Важно

- ⚠️ Telegram требует **HTTPS** для webhook. Для localhost используйте **ngrok** или **localtunnel**.
- 🔐 Не храните токен бота в публичных репозиториях — он сохраняется в БД через админку.
- 📊 Все отправленные уведомления логируются в таблице `telegram_notifications_log`.
